图片格式分类
jpg:有损压缩，体积小，加载快，不支持透明（缺点：处理矢量图形，颜色对比比较明显的图片等时会图片模糊）
png:无损压缩，质量高，体积大，支持透明（缺点：体积大）
svg:文本文件，体积小，不失真，兼容性比较好（缺点：渲染成本比较高）
base64:文本文件，可以编程，作为小图标解决方案而存在的。（类似于雪碧图的补充）
webp:像jpg一样对丰富的图片信手拈来，像png一样支持透明，像gif一样支持动态（兼容性问题）

知识扩展：
1.雪碧图，被用到众多使用大量小图标的网页应用之上，可取图像的一部分使用，使得一个图像文件替代多个小文件成为可能，
相较于使得使用一个图像文件替代多个小文件成为可能。相比于一个小图标一个图像文件，单独一张图所需要的http请求更少，更加友好。

2.Base64 是一种用于传输 8Bit 字节码的编码方式，通过对图片进行 Base64 编码，
我们可以直接将编码结果写入 HTML 或者写入 CSS，从而减少 HTTP 请求的次数。那能不能把大图也用base64转码呢？？
  不可以，Base64 编码后，图片大小会膨胀为原文件的 4/3（这是由 Base64 的编码原理决定的）。如果我们把大图也编码到 HTML 或
CSS 文件中，后者的体积会明显增加，即便我们减少了 HTTP 请求，也无法弥补这庞大的体积带来的性能开销，得不偿失。
